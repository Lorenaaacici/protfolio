Project Overview

Brisentinel

"Brisentinel" is a platform that integrates property listings with essential risk assessment data, such as flood zones and crime rates, to help users make informed real estate decisions. The platform features an intuitive interface that allows users to explore properties through an interactive map.

By visualizing key data points, "Brisentinel" aims to enhance user decision-making and prioritize accessibility. This portfolio outlines the design and development process, covering key challenges, iterations, and design principles used to create a user-centered and functional platform.

Process & Evidence

4A: Ideation and Research

Progress and Contribution

Group Progress: During this phase, our team brainstormed potential solutions for the gaps in existing real estate platforms. Together, we developed the initial design concept and identified key datasets, such as bushfire, flood, and crime data, that would inform our project. The group collectively decided to use an interactive map as the primary tool for risk assessment.

My Contribution: I conducted an in-depth evaluation of the selected datasets to assess their feasibility for integration. I proposed the idea of using an interactive map to visualize the data effectively. Additionally, I created documentation summarizing user needs and proposed features based on the group discussions.

4B: Design and Prototyping

Progress and Contribution

Group Progress: In this phase, our team developed the initial wireframes and sketches of the platform's user interface. We explored various design layouts and discussed user-centered design principles to ensure a smooth user experience. The group reviewed the wireframes and provided feedback for further refinement.

My Contribution: I took the lead in creating detailed wireframes using Figma, focusing on designing the layout for the interactive map and property listings. I also sketched user flow diagrams to demonstrate how users would interact with different platform features.

4C: MVP Development

Progress and Contribution

Group Progress: During the MVP development, our team worked on integrating the interactive map using the Leaflet API. We collaborated on coding basic functionalities, testing the map's zoom, pan, and data filtering features.

My Contribution: My primary responsibility was coding the map's functionalities and developing JavaScript functions for real-time data filtering. I also implemented an MVC (Model-View-Controller) architecture to improve the separation of data management and the user interface.

4D: Final Prototype and Refinements

Progress and Contribution

Group Progress: In this phase, our team conducted user testing to gather feedback, focusing on areas such as map responsiveness, data visualization, and accessibility. We discussed key refinements based on the feedback.

My Contribution: I was responsible for implementing the carousel-style display for saved properties on the profile page and improving the map's responsiveness. I also integrated accessibility features, such as keyboard navigation and enhanced color contrast.
